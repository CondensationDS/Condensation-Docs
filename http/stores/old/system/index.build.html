Condensation is a distributed data system, in which users (and services) store and share their data through Condensation stores:

On each device, a user (or service) keeps a actor, consisting of a key pair, and optional public information. On each store, a user (or service) has an account for each of his/her identities.

Data is serialized as immutable objects and trees, and protected through end-to-end encryption.

This section of the documentation contains a high-level description of Condensation. To find out how to use Condensation on a specific storage or communication system, check out the protocols section.
